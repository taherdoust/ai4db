# Secure API Key Setup for AI4DB

## üîí Security First

This guide shows you how to use OpenRouter API securely without exposing your API key in the GitHub repository.

---

## üìã Quick Setup (3 Steps)

### **Step 1: Get Your API Key**

1. Go to https://openrouter.ai/
2. Sign up or log in
3. Navigate to "Keys" section
4. Create a new API key
5. Copy the key (starts with `sk-or-v1-...`)

### **Step 2: Set Up .env File (Recommended)**

```bash
# Navigate to project directory
cd ~/Desktop/ai4db

# Copy the example file
cp .env.example .env

# Edit .env file and add your actual API key
nano .env
```

In the `.env` file, replace `sk-or-v1-your-key-here` with your actual key:

```bash
# AI4DB Configuration File
OPENROUTER_API_KEY=sk-or-v1-YOUR-ACTUAL-KEY-HERE
OPENROUTER_MODEL=openai/gpt-4-turbo-preview
```

**Save and exit** (Ctrl+X, then Y, then Enter in nano)

### **Step 3: Verify .env is Protected**

```bash
# Check that .env is in .gitignore
cat .gitignore | grep ".env"

# Should show:
# .env
# .env.local
```

‚úÖ Your API key is now secure and won't be committed to GitHub!

### **Method 1: Environment Variable (Temporary)**

### **Method 2: Add to .bashrc (Permanent)**

```bash

# Add at the end:

# Save and reload

```

‚ö†Ô∏è **Warning:** This exposes the key in your .bashrc file. Use .env method instead.

## üõ°Ô∏è Security Best Practices

### **DO:**
‚úÖ Use `.env` file for API keys
‚úÖ Keep `.env` in `.gitignore`
‚úÖ Use `.env.example` as a template (without real keys)
‚úÖ Use `python-dotenv` library to load `.env` automatically

### **DON'T:**
‚ùå Hardcode API keys in Python files
‚ùå Commit `.env` to GitHub
‚ùå Share your API key in chat/email
‚ùå Use the same key for multiple projects (create separate keys)

## üêõ Troubleshooting

### **Problem: API key not found**

```bash
‚ö†Ô∏è  API key not found. Please set OPENROUTER_API_KEY
```

**Solution:**
1. Check if `.env` file exists: `ls -la .env`
2. Check if `python-dotenv` is installed: `pip install python-dotenv`
3. Or use environment variable: `export OPENROUTER_API_KEY="your-key"`

### **Problem: .env file not loading**

**Solution:**
```bash
# Install python-dotenv
pip install python-dotenv

# Verify installation
python -c "from dotenv import load_dotenv; print('‚úì python-dotenv installed')"
```

### **Problem: API key in git status**

```bash
# If .env shows up in git status
git rm --cached .env
git add .gitignore
git commit -m "Protect API keys"
```
